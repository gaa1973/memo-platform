# 📦 06. コアコンポーネント

このセクションでは、アプリケーションの核となる主要なクラスや関数について解説します。

## 🚀 バックエンド (`auth-api`)

### `AuthController` (`auth.controller.ts`)

ユーザー認証（サインアップ、ログイン、トークン検証）を担当するコントローラーです。

- `signup`: 新規ユーザーの作成とパスワードハッシュ化。
- `login`: 資格情報の検証と JWT の発行。
- `verify`: 提供されたトークンが有効かどうかのチェック。

### `MemosController` (`memos.controller.ts`)

メモの CRUD 操作を管理します。

- すべてのメソッドは認証ミドルウェアを通過していることが前提となります。
- `createMemo`: ログインユーザーに紐づく新しいメモを保存。
- `getMemos`: ユーザー自身のメモ一覧のみを取得。

### `PrismaClient` (`db/index.ts` 等)

データベースへの型安全なアクセスを提供します。

## ⚛️ フロントエンド (`my-login-app`)

### `App` コンポーネント (`App.tsx`)

アプリケーションのルートコンポーネントであり、以下の役割を担います。

- **ルーティング**: ログイン状態に応じた表示の切り替え。
- **グローバルステート**: 認証済ユーザー情報の保持。

### `LoginForm` / `SignupForm` (`components/`)

ユーザー入力を受け取り、バリデーション（Zod）を行ってから API を呼び出します。

- `react-hook-form` を使用して、再レンダリングを最適化しつつ入力を管理。

### `useAuth` カスタムフック (`hooks/`)

認証に関連するロジック（ログイン、ログアウト、トークン確認）を抽象化し、各コンポーネントから簡単に利用できるようにします。
