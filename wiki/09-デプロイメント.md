# 🚢 09. デプロイメント

Memo Platform アプリケーションを本番環境（またはステージング環境）へデプロイする手順について解説します。

## 🏗 ビルド手順

### バックエンド (`auth-api`)

バックエンドは TypeScript で記述されているため、JavaScript にコンパイルする必要があります。

```bash
cd auth-api
# ビルド実行 (現在は direct execution を想定しているが、以下の手順が一般的)
# npm run build
```

### フロントエンド (`my-login-app`)

```bash
cd my-login-app
npm run build
# `build/` ディレクトリに静的ファイルが生成されます
```

## ☁️ デプロイメント先（例）

### フロントエンド

- **Vercel / Netlify**: `build/` フォルダをアップロードするだけで簡単にデプロイ可能です。
- **S3 + CloudFront**: 静的ホスティングとして利用可能です。

### バックエンド

- **Render / Heroku**: Node.js アプリケーションとしてデプロイできます。
- **Docker**: `Dockerfile` を作成し、任意のサーバー（AWS EC2, Google Cloud Run 等）でコンテナとして実行するのが推奨されます。

## 🛠 本番環境での設定

- **環境変数**: `DATABASE_URL` と `JWT_SECRET` を必ず本番用の秘匿値に変更してください。
- **HTTPS**: 認証トークン（Cookie）を安全に扱うため、必ず HTTPS 環境で運用してください。
- **CORS**: フロントエンドのドメインのみ許可するように `CORS` 設定を制限してください。
