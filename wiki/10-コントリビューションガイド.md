# 🤝 10. コントリビューションガイド

Memo Platform プロジェクトへの貢献を歓迎します！開発プロセスを円滑に進めるためのガイドラインを以下に示します。

## 🌲 ブランチ戦略

- **main**: 本番環境用コード。常に安定した状態を維持します。
- **develop**: 開発のベースとなるブランチ。
- **feature/xxx**: 個別の機能追加や修正のためのブランチ。

## 🛠 開発の流れ

1. `develop` ブランチから最新コードを取得し、新しい `feature` ブランチを作成します。
2. コードを変更し、ローカルでテストを実行します (`npm test`)。
3. コードスタイルを確認します (`npm run lint` / `npm run format`)。
4. 変更をコミットし、プルリクエストを作成します。
5. レビューを受け、承認されたら `develop` にマージされます。

## 📜 コーディング規約

- **TypeScript**: 明示的な型定義を推奨します。`any` の使用は避けてください。
- **ESLint / Prettier**: 設定ファイルに従い、自動整形を適用してください。
- **コミットメッセージ**: 英語または日本語で、変更内容が明確に伝わるように記述してください。
  - 例: `feat: ログインバリデーションの強化`, `fix: メモ削除時のバグ修正`

## 💬 報告とフィードバック

バグを発見した場合や改善の提案がある場合は、Issue を作成して報告してください。
