# 🧪 08. テスト戦略

Memo Platform では、品質維持のために自動テストを導入しています。

## 🛠 使用ツール

- **Jest**: 基本的なテストランナー。
- **ts-jest**: TypeScript のテスト実行用。
- **React Testing Library**: フロントエンドコンポーネントのテスト用。
- **Prisma**: テスト用データベースの状態管理（クリーンアップ等）。

## 🚀 バックエンドテスト (`auth-api`)

バックエンドでは、API エンドポイントの統合テストを重視しています。

### テストの実行

```bash
cd auth-api
npm test
```

### 主要なテスト項目

- **認証フロー**: ユーザー登録 → ログイン → 認証が必要なページへのアクセスが正しく行えるか。
- **バリデーション**: 無効な形式のメールアドレスや、短すぎるパスワードを拒否するか。
- **データベース操作**: メモの保存・取得・削除が正しく DB に反映されるか。

## ⚛️ フロントエンドテスト (`my-login-app`)

フロントエンドでは、ユーザーインタラクションと正しい情報の表示をテストします。

### テストの実行

```bash
cd my-login-app
npm test
```

### 主要なテスト項目

- **コンポーネントのレンダリング**: ログインフォームが正しく表示されるか。
- **ユーザー入力の反映**: 入力フィールドに文字を入力した際、エラーメッセージが動的に表示されるか（Zod バリデーション）。
- **ログイン状態の遷移**: ログイン成功後に `MemoApp` コンポーネントに切り替わるか。
